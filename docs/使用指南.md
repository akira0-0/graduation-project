## 项目简介

本项目是一个多平台社交媒体爬虫系统，支持小红书、微博等平台的数据采集。所有平台数据统一为标准格式，便于后续分析和处理。

### 主要功能

- ✅ 小红书内容爬取（关键词搜索、笔记详情、评论）
- ✅ 微博内容爬取（关键词搜索、帖子、评论）
- ✅ 统一数据格式（跨平台数据标准化）
- ✅ 数据去重与增量保存（按日期自动合并）
- ✅ 支持IP代理池（防封禁）
- ✅ CDP模式（使用真实浏览器，提高反检测能力）

### 支持平台

| 平台 | 状态 | 数据格式 |
|------|------|----------|
| 小红书 | ✅ 支持 | 旧格式（需转换） |
| 微博 | ✅ 支持 | 统一格式 ✨ |
| 抖音 | ⚠️ 代码存在但未启用 | - |
| B站 | ⚠️ 代码存在但未启用 | - |
| 知乎 | ⚠️ 代码存在但未启用 | - |

## 微博爬虫配置与使用

### 📁 位置

微博爬虫是一个独立的子项目：`weibo_crawler/`

### 配置步骤

#### 1. 创建配置文件

进入微博爬虫目录，配置文件应在：

```
weibo_crawler/weibo_crawler/config.json
```

配置文件示例：

```json
{
  "cookie": "YOUR_WEIBO_COOKIE_HERE",
  "keywords": ["人工智能", "Python", "机器学习"],
  "max_pages": 3
}
```

#### 2. 获取微博 Cookie

**方法1: Chrome浏览器（推荐）**

1. 打开 [https://m.weibo.cn](https://m.weibo.cn)
2. 登录你的微博账号
3. 按 `F12` 打开开发者工具
4. 切换到 `Network` (网络) 标签
5. 刷新页面，找到任意请求
6. 在 `Request Headers` 中找到 `Cookie` 字段
7. 复制整个 Cookie 值（很长的一串）
8. 粘贴到 `config.json` 的 `cookie` 字段

**Cookie 示例**（已脱敏）：

```
SUB=_2A25K...; SUBP=001h...; MLOGIN=1; M_WEIBOCN_PARAMS=...
```

⚠️ **注意**：
- Cookie 包含敏感信息，不要分享给他人
- Cookie 有有效期，失效后需要重新获取
- 不要将 `config.json` 提交到 Git 仓库

#### 3. 配置参数说明

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `cookie` | 字符串 | 微博登录Cookie（必需） | `"SUB=_2A25K..."` |
| `keywords` | 数组 | 搜索关键词列表 | `["人工智能", "Python"]` |
| `max_pages` | 数字 | 每个关键词爬取的最大页数 | `3` |

## 小红书爬虫配置与使用

### 📁 位置

小红书爬虫是主项目的一部分，配置文件：`config/base_config.py`

### 配置步骤

#### 1. 修改配置文件

编辑 `config/base_config.py`：

```python
# 基础配置
PLATFORM = "xhs"  # 平台选择
KEYWORDS = "人工智能,Python,机器学习"  # 关键词（英文逗号分隔）
LOGIN_TYPE = "qrcode"  # 登录方式: qrcode(二维码) | cookie(Cookie)
COOKIES = ""  # 如果使用cookie登录，填写这里
CRAWLER_TYPE = "search"  # 爬取类型: search(搜索) | detail(详情) | creator(创作者)

# CDP模式配置（推荐开启）
ENABLE_CDP_MODE = True  # 使用真实浏览器，反检测更强
CDP_DEBUG_PORT = 9222  # CDP调试端口
CDP_HEADLESS = False  # 是否无头模式

# 代理配置（可选）
ENABLE_IP_PROXY = False  # 是否启用IP代理
IP_PROXY_POOL_COUNT = 2  # 代理池数量
IP_PROXY_PROVIDER_NAME = "kuaidaili"  # 代理提供商

# 数据保存配置
SAVE_DATA_OPTION = "json"  # 保存格式: json | csv | excel
```

#### 2. 其他重要配置

**小红书搜索配置** (`config/xhs_config.py`):

```python
# 爬取数量限制
CRAWLER_MAX_NOTES_COUNT = 20  # 最多爬取笔记数

# 排序方式
SORT_TYPE = "general"  # general(综合) | popularity_descending(热度) | time_descending(最新)

# 评论配置
ENABLE_GET_COMMENTS = True  # 是否爬取评论
ENABLE_GET_SUB_COMMENTS = False  # 是否爬取子评论
CRAWLER_MAX_COMMENTS_COUNT_SINGLENOTES = 10  # 单条笔记最多爬取评论数
```

#### 3. 登录方式

**方式1: 二维码登录（推荐）**

```python
LOGIN_TYPE = "qrcode"
```

运行爬虫后会自动弹出浏览器，用小红书APP扫码登录。

**方式2: Cookie登录**

1. 打开小红书网页版：[https://www.xiaohongshu.com](https://www.xiaohongshu.com)
2. 登录账号
3. 按 `F12` 打开开发者工具
4. 切换到 `Application` → `Cookies`
5. 复制所有 Cookie（或关键Cookie如 `web_session`）
6. 粘贴到配置文件：

```python
LOGIN_TYPE = "cookie"
COOKIES = "web_session=xxx; ..."
```

## 数据格式转换

### 为什么需要转换？

小红书爬虫输出的是**旧格式**，字段名不统一（如 `note_id`, `user_id`, `liked_count`），与微博的统一格式不一致。转换后可以：

- ✅ 统一字段名（`id`, `author.id`, `metrics.likes`）
- ✅ 统一数据类型（"6.8万" → 68000）
- ✅ 便于跨平台分析
- ✅ 便于导入数据库

### 转换方法

#### 使用迁移脚本

```bash
# 运行迁移脚本
uv run scripts\migrate_data_format.py
```

## 数据库导入

### 数据库表结构设计

根据统一格式的JSON数据，推荐以下表结构：

#### 1. 帖子表 (posts) - PostgreSQL

```sql
CREATE TABLE IF NOT EXISTS posts (
    -- 主键和基础信息
    id VARCHAR(50) PRIMARY KEY,
    platform VARCHAR(20) NOT NULL,
    type VARCHAR(20),
    url TEXT,
    title VARCHAR(500),
    content TEXT,
    publish_time TIMESTAMP,
    last_update_time TIMESTAMP,
    
    -- 作者信息
    author_id VARCHAR(50),
    author_nickname VARCHAR(100),
    author_avatar VARCHAR(500),
    author_is_verified BOOLEAN DEFAULT FALSE,
    author_ip_location VARCHAR(50),
    
    -- 媒体信息（使用JSONB存储数组）
    media_images JSONB,
    media_video_url TEXT,
    
    -- 互动数据
    metrics_likes INTEGER DEFAULT 0,
    metrics_collects INTEGER DEFAULT 0,
    metrics_comments INTEGER DEFAULT 0,
    metrics_shares INTEGER DEFAULT 0,
    
    -- 其他元数据
    tags JSONB,
    source_keyword VARCHAR(100),
    task_id VARCHAR(50),
    crawl_time BIGINT,
    
    -- 扩展字段（平台特有数据）
    extra JSONB,
    
    -- 系统字段
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 添加表注释
COMMENT ON TABLE posts IS 'Posts/Content data table';
COMMENT ON COLUMN posts.id IS 'Unique content ID';
COMMENT ON COLUMN posts.platform IS 'Platform identifier (xhs/weibo/douyin/bilibili/zhihu)';
COMMENT ON COLUMN posts.type IS 'Content type (video/image/text)';
COMMENT ON COLUMN posts.media_images IS 'Image URL array (JSONB format)';
COMMENT ON COLUMN posts.extra IS 'Extended fields for platform-specific data';

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_posts_platform ON posts(platform);
CREATE INDEX IF NOT EXISTS idx_posts_author_id ON posts(author_id);
CREATE INDEX IF NOT EXISTS idx_posts_publish_time ON posts(publish_time);
CREATE INDEX IF NOT EXISTS idx_posts_source_keyword ON posts(source_keyword);
CREATE INDEX IF NOT EXISTS idx_posts_task_id ON posts(task_id);
CREATE INDEX IF NOT EXISTS idx_posts_platform_author ON posts(platform, author_id);

-- 创建自动更新时间戳的触发器
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_posts_updated_at BEFORE UPDATE ON posts
    FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

#### 2. 评论表 (comments) - PostgreSQL

```sql
CREATE TABLE IF NOT EXISTS comments (
    -- 主键和基础信息
    id VARCHAR(50) PRIMARY KEY,
    content_id VARCHAR(50) NOT NULL,
    platform VARCHAR(20) NOT NULL,
    content TEXT,
    publish_time TIMESTAMP,
    
    -- 作者信息
    author_id VARCHAR(50),
    author_nickname VARCHAR(100),
    author_avatar VARCHAR(500),
    author_ip_location VARCHAR(50),
    
    -- 互动数据
    metrics_likes INTEGER DEFAULT 0,
    metrics_sub_comments INTEGER DEFAULT 0,
    
    -- 评论层级关系
    parent_comment_id VARCHAR(50),
    root_comment_id VARCHAR(50),
    reply_to_user_id VARCHAR(50),
    reply_to_user_nickname VARCHAR(100),
    comment_level INTEGER DEFAULT 1,
    
    -- 其他元数据
    task_id VARCHAR(50),
    crawl_time BIGINT,
    
    -- 扩展字段
    extra JSONB,
    
    -- 系统字段
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 添加表注释
COMMENT ON TABLE comments IS 'Comments data table';
COMMENT ON COLUMN comments.id IS 'Unique comment ID';
COMMENT ON COLUMN comments.content_id IS 'Associated content ID (links to posts table)';
COMMENT ON COLUMN comments.parent_comment_id IS 'Parent comment ID (NULL for top-level comments)';
COMMENT ON COLUMN comments.root_comment_id IS 'Root comment ID (always points to top-level comment)';
COMMENT ON COLUMN comments.comment_level IS 'Comment level (1=top-level, 2=reply, 3=nested reply...)';

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_comments_content_id ON comments(content_id);
CREATE INDEX IF NOT EXISTS idx_comments_platform ON comments(platform);
CREATE INDEX IF NOT EXISTS idx_comments_author_id ON comments(author_id);

## 常见问题

### Q1: 微博爬虫一直返回空数据

**原因**：Cookie失效或未配置

**解决**：
1. 检查 `config.json` 中的 `cookie` 是否正确
2. 重新登录微博获取新的Cookie
3. 确保Cookie包含 `SUB`, `SUBP` 等关键字段

### Q2: 小红书扫码登录一直失败

**原因**：滑动验证码未通过

**解决**：
1. 设置 `HEADLESS = False` 打开浏览器窗口
2. 手动滑动验证码
3. 再扫码登录

### Q3: 数据转换后找不到文件

**原因**：路径配置错误

**解决**：
- 检查 `data/xhs/json/` 是否有数据
- 确认转换后的数据在 `data/unified/xhs/`
- 查看脚本输出的路径信息

### Q4: 导入数据库时中文乱码

**原因**：字符集不是 UTF-8

**解决**：
1. 确保数据库字符集为 `utf8mb4`
2. 表字符集也设置为 `utf8mb4`
3. 连接时指定 `charset='utf8mb4'`

### Q5: 爬虫速度太慢

**原因**：默认有延迟防止被封

**解决**：
- 调整 `config/base_config.py` 中的 `CRAWLER_MAX_SLEEP_SEC`
- 启用IP代理池 `ENABLE_IP_PROXY = True`
- 增加并发数 `MAX_CONCURRENCY_NUM`

⚠️ **警告**：过快爬取可能导致账号被封

### Q6: uv sync 失败

**原因**：Python版本不兼容

**解决**：
1. 检查Python版本：`python --version`
2. 确保是 3.11 或 3.12
3. 如果是 3.13+，需要降级到 3.12

```bash
# 使用uv指定Python版本
uv python install 3.12
uv venv --python 3.12
```

### Q7: 统一格式数据的 likes 数值不准确

**原因**：小红书原始数据是字符串"6.8万"

**解决**：
- 转换器会自动处理："6.8万" → 68000
- 检查 `tools/data_format_converter.py` 的 `parse_count()` 函数
- 如果有特殊格式，需要修改解析逻辑

### 核心流程

**小红书爬虫流程**：

```
main.py
  → media_platform/xhs/core.py (爬取)
  → store/xhs/__init__.py (数据处理)
  → store/xhs/_store_impl.py (保存)
  → data/xhs/json/*.json (旧格式输出)
  
  → tools/data_format_converter.py (转换)
  → data/unified/xhs/*.json (统一格式输出)
```

**微博爬虫流程**：

```
weibo_crawler/main.py
  → weibo_crawler/crawler.py (爬取)
  → weibo_crawler/utils.py (转换为统一格式)
  → weibo_crawler/file_writer.py (保存)
  → data/unified/weibo/*.json (统一格式输出)
```
